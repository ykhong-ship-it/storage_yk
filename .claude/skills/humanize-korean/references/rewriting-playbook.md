---

## 6대 원칙

1. **의미 불변성** - 수치·사실·고유명사·직접인용은 단 한 글자도 바꾸지 않는다
2. **톤 유지** - 원문이 구어체면 구어체로, 칼럼이면 칼럼으로 유지
3. **국소성** - 탐지된 span만 수정; 깨끗한 구간은 손대지 않는다
4. **자연성 > 완벽성** - 문법적으로 완벽한 문장보다 실제로 자연스러운 문장
5. **변경률 경보** - 레벤슈타인 거리 기준 30% 초과 시 경고, 50% 초과 시 중단
6. **증거 기반** - 모든 수정에 taxonomy ID(예: A-1, D-2) 근거 명시

## 카테고리별 처방

### A. 번역투 제거

| 원문 패턴 | 처방 |
|-----------|------|
| "~에 대해 논의한다" | "~를 논의한다" |
| "~를 통해 X한다" | "~로 X한다" |
| "~에 기반하여" | "~를 바탕으로" |
| "그/그녀가 말했다" | "[이름]/[직함]이 말했다" |
| 관계절 3중 중첩 | 문장 분리: "A다. 이를 바탕으로 B한다." |

### B. 영어 인용 처리

- 전문 용어: 최초 1회만 한국어(English) 병기, 이후 한국어만
- 일반 단어: 한국어 번역 제공 (leverage → 활용)

### C. 구조 개선

기계적 나열을 산문으로:
- Before: 첫째, A다. 둘째, B다. 셋째, C다.
- After: A하며, B하는 동시에 C한다.

불릿을 문장으로:
- Before: X는 효율적이다 / Y는 빠르다 / Z는 저렴하다
- After: X는 효율적이고 빠르며 저렴하다.

### D. AI 관용구 삭제

| 삭제 대상 | 처리 방식 |
|-----------|-----------|
| "결론적으로" | 삭제 후 직접 서술 |
| "매우 중요하다" | 왜 중요한지 구체화하거나 단순화 |
| "다양한 측면에서" | 실제 측면 명시 또는 삭제 |
| "획기적인/혁신적인" | 수치·사례로 대체 |

### E. 리듬 회복

- 짧은 문장(10자 내외)과 긴 문장(40자 이상)을 의도적으로 혼합
- "~하고 있다" → 상태나 일반적 사실이면 "~한다"
- "~다" 100% → "~이다", "~겠다", "~리라" 혼합

### F. 수식어 정리

- 유의어 중복: "새롭고 신선한" → "신선한"
- -적 스택: "효율적이고 체계적이며 합리적인" → 핵심 1개 선택
- 관형어 3중 이상: 문장 재구성

### G. 완곡 표현 정리

- Before: 효과가 있을 수도 있을 것으로 예상될 수 있다
- After: 효과가 있을 것으로 예상된다

### H. 접속사 정리

- "또한" 3회 이상: 3회째부터 삭제 또는 문장 연결로 흡수
- "따라서": 실제 인과관계 없으면 삭제
- "그리고, X" → "그리고 X" (쉼표 제거)

### I. 형식명사 정리

| 원문 | 처방 |
|------|------|
| "중요한 것이다" | "중요하다" |
| "주목할 점이다" | "주목해야 한다" |
| "분석하는 데 있어" | "분석할 때" |

### J. 시각 장식 정리

- 볼드: 문단 내 진짜 핵심 1~2곳만 유지
- 강조 따옴표: 전부 제거
- em-dash 과다: 쉼표 또는 괄호로 교체

## 변경률 기준

| 범위 | 판정 |
|------|------|
| 0-5% | 편집량 부족 |
| 5-30% | 권장 범위 |
| 30-50% | 경고 - fidelity 점검 필수 |
| 50% 초과 | 중단 - 원본으로 롤백 |

## 15개 최종 점검 항목

1. [ ] 수치·날짜·고유명사 변경 없음
2. [ ] 직접인용 내용 변경 없음
3. [ ] S1 패턴 전수 제거 확인
4. [ ] S2 패턴 고밀도 구간 처리 확인
5. [ ] 변경률 5-30% 범위 내
6. [ ] 원문 글의 종류 유지
7. [ ] 격식 수준(구어/문어) 유지
8. [ ] 영어 병기 1회 규칙 준수
9. [ ] 접속사-쉼표 결합(C-11) 전수 확인
10. [ ] Hedging 체인(G-3) 해체 확인
11. [ ] "그/그녀" 직역(A-16) 전수 제거
12. [ ] "결론적으로"(D-1) 전수 제거
13. [ ] "또한" 문두 3회 이상(H-1) 처리
14. [ ] 이모지(C-4) 제거 완료
15. [ ] 전체 자연스럽게 소리 내어 읽기 통과
